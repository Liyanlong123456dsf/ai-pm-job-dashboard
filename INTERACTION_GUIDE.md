# AI 岗位全景分析 Dashboard — 交互说明文档

> **版本**：v2.0 | **线上地址**：https://ai-pm-job-dashboard.netlify.app  
> **GitHub**：https://github.com/Liyanlong123456dsf/ai-pm-job-dashboard

---

## 一、页面结构总览

页面为单页应用（SPA），由 **顶部导航 + 5 个内容板块 + 悬浮 AI 对话面板** 组成。

| 板块 | 锚点 ID | 背景色 | 核心功能 |
|---|---|---|---|
| 总览 Hero | `#hero` | 纯黑 | KPI 数字总览，点击可跳转对应板块 |
| 十大方向 | `#directions` | 深灰 | 方向卡片 + 关键词筛选 + 岗位列表 |
| 薪资图谱 | `#salary` | 深黑 | 散点图 + 薪资梯队筛选 |
| 城市对比 | `#cities` | 亮白 | 横向柱状图 + 城市筛选 |
| 自由探索 | `#explore` | 纯黑 | 方向 × 城市 × 薪资 × 经验 四维筛选 |

---

## 二、导航栏 (Nav)

- **固定在页面顶部**，高 52px，毛玻璃背景
- 包含 Logo「AI Jobs」和 5 个锚点链接：总览、方向、薪资、城市、探索
- **滚动自动高亮**：页面滚动时，当前可视区域对应的导航链接自动添加蓝色下划线
- **点击平滑滚动**：点击任意导航链接，页面平滑滚动到对应板块
- **音效**：点击时播放 `click` 音效

## 三、总览 Hero（#hero）

### 3.1 标题与副标题
- 大标题「AI 岗位**全景洞察**」，渐变蓝色高亮
- 副标题列出覆盖的方向和城市

### 3.2 KPI 数字卡片
四个核心指标，横向排列：

| KPI | 说明 | 点击跳转 |
|---|---|---|
| 相关岗位数 | 数据总量 | → 探索板块 |
| 细分方向数 | 方向类别数 | → 方向板块 |

- **数据更新时间戳**：显示在 KPI 下方

## 四、十大方向（#directions）

### 4.1 方向卡片网格
- **5列网格**布局（移动端 2 列）
- 每张卡片显示：方向名称、岗位数量、均薪
- **点击卡片**：选中/取消选中该方向，触发下方列表和关键词筛选刷新
- **选中态**：蓝色边框 + 上浮 4px + 蓝色光晕
- **音效**：`toggle`

### 4.2 关键词药丸（Pills）
- 根据当前选中方向动态生成 Top 10 高频关键词
- 第一项固定为「全部」
- **点击药丸**：按关键词二次筛选岗位列表
- **选中态**：蓝色实底

### 4.3 岗位列表
- 卡片网格展示（最多 300px 宽，自适应列数）
- 默认显示 12 条，点击「加载更多」按钮每次追加 12 条
- **点击岗位卡片**：打开详情弹窗

## 五、薪资图谱（#salary）

### 5.1 气泡散点图
- **横轴**：城市分布
- **纵轴**：等效月薪（K）

- **悬停**：显示 Tooltip（岗位名、公司、城市、薪资）
- **点击气泡**：打开岗位详情弹窗
- 上方有颜色图例说明

### 5.2 薪资梯队筛选
- 药丸组：全部 / 50K+ / 30-50K / 15-30K / 8-15K / <8K
- **点击**：筛选对应薪资区间的岗位列表

### 5.3 岗位列表
- 同方向板块，12 条分页 + 加载更多

## 九、岗位详情弹窗（Modal）

点击任意岗位卡片或散点图气泡触发：

```
┌──────────────────────────────── ✕ ┐
│ 岗位标题                          │
│ 60-90K·15薪 (≈75K/月)             │  ← 薪资（蓝色加粗）
│ 公司 · 城市 · 5-10年 · 本科       │  ← 基本信息
│ [方向标签] [关键词标签×5]          │  ← 分类标签
│ ┌─────────────────────────────┐   │
│ │ 职位描述（可滚动，最高240px）│   │  ← 灰底圆角框，支持上下滚动
│ └─────────────────────────────┘   │
│ 同城同方向推荐 →                   │  ← 最多4条相关推荐
│   推荐岗位1                        │
│   推荐岗位2                        │
│ [🤖 问 AI 分析这个岗位]            │  ← 蓝色渐变按钮
└───────────────────────────────────┘
```

### 交互说明
- **关闭方式**：✕ 按钮 / 点击遮罩层 / 按 Esc 键
- **推荐卡片**：点击直接切换到该岗位详情
- **问 AI 按钮**：关闭弹窗 → 打开聊天面板 → 自动发送分析请求
- **音效**：打开 `modalOpen`，关闭 `modalClose`
- **动画**：遮罩淡入 + 弹窗从下方滑入

## 十、AI 求职顾问（聊天面板）

### 10.1 悬浮按钮（FAB）
- 固定在页面右下角，蓝色圆形，聊天图标
- **点击**：打开/关闭聊天面板
- 打开时图标旋转 90°
- **音效**：`slide`

### 10.2 聊天面板
- 从屏幕右侧滑入，宽 420px（移动端全屏宽）
- 结构：设置层 → 头部 → 消息区域 → 输入框

### 10.3 头部
| 元素 | 功能 |
|---|---|
| 绿色圆点 + 标题 | 「AI 求职顾问」在线状态指示 |
| API Badge | 显示当前接口（扣子/Kimi），**点击一键切换** |
| 🗑 按钮 | 清空对话历史 |
| ⚙ 按钮 | 打开 API 设置 |
| ✕ 按钮 | 关闭面板 |

### 10.4 API 设置面板

支持两个 AI 接口，通过 Tab 切换：

#### 扣子 Coze
- Token 输入框（`pat_xxx` 格式）
- Bot ID 输入框
- 直连 `api.coze.cn/v3/chat`（SSE 流式）

#### Kimi
- API Key 输入框（`sk-kimi-xxx` 或 `sk-xxx`）
- `sk-kimi-` 开头：Kimi Code Key → 本地走 `api.kimi.com` 直连，线上走 Netlify 代理（`/api/kimi/messages`）
- `sk-` 开头：Moonshot Key → 直连 `api.moonshot.ai`

**数据存储**：所有凭证保存在浏览器 `localStorage`，不上传服务器。

### 10.5 欢迎消息与快捷问题
首次打开显示：
- 欢迎语 + 数据概览
- 4 个快捷问题按钮（点击直接发送）：
  1. 「我是3年经验的产品经理，想转AI方向...」
  2. 「视频AI和电商AI哪个薪资更高？」
  3. 「杭州和上海的AI PM岗位有什么区别？」
  4. 「想拿30K+月薪需要什么条件？」

### 10.6 消息交互
- **用户消息**：蓝色气泡，右对齐
- **AI 回复**：深灰气泡，左对齐，支持 Markdown 渲染（表格、列表、加粗、代码块等）
- **发送方式**：点击发送按钮 / Enter 发送 / Shift+Enter 换行
- **输入框自适应**：随内容高度自动扩展，最高 120px
- **流式输出**：实时逐字显示 AI 回复（SSE 协议）
- **历史记录**：保留最近 20 轮对话作为上下文
- **音效**：发送 `send`，收到回复 `receive`

### 10.7 错误处理
| 错误类型 | 提示内容 |
|---|---|
| 401 认证失败 | 提示检查 Key 格式和有效期 |
| 429 频率限制 | 提示等待后重试 |
| CORS 跨域 | 提示切换接口或部署方式 |
| 网络错误 | 显示具体错误信息 |

## 十二、互动音效系统

基于 **Web Audio API** 实时合成，无需加载外部音频文件：

| 音效名 | 触发场景 | 声音特征 |
|---|---|---|
| `click` | 卡片点击、按钮点击、KPI点击、加载更多 | 轻柔下降音 (1200→800Hz) |
| `toggle` | 药丸筛选、方向卡片、柱状图点击 | 短促滴答 (1000Hz) |
| `modalOpen` | 弹窗打开 | 上升音 (400→900Hz) |
| `modalClose` | 弹窗关闭 | 下降音 (800→350Hz) |
| `slide` | 聊天面板开关 | 柔滑滑动 (500→700Hz) |
| `send` | 发送消息 | 清脆短音 (880→1320Hz) |
| `receive` | 收到AI回复 | 双音通知 (660+880Hz) |

所有音效音量极低（0.04-0.08），持续时间 40-150ms，不干扰使用。

## 十五、数据流

```
jobs_data.json (887条)
      ↓ fetch + 缓存破坏(?t=时间戳)
    DATA 数组
      ↓
  initAll() → 5个render函数并行渲染
      ↓
  用户交互 → 筛选/过滤 → 重新渲染对应板块
```

- 数据来源：BOSS直聘爬虫（`boss_dp.py`）→ 清洗管道（`pipeline.py`）→ 合并（`merger.py`）→ `jobs_data.json`
- 前端纯静态，无后端依赖
- AI 对话通过浏览器直接调用第三方 API（扣子/Kimi/Moonshot）
